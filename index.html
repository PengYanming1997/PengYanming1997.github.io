1685. 有序数组中差绝对值之和. Medium

利用数学规律求解 解题思路 : （1）如果题目中求的不是差的绝对值的和 而是差的绝对值的和 那么ans[i] = nums[i] * n - sum(n)； （2）由于题目中给出的数组是一个非递减数组 所以 nums[i]与其之前的元素的差是大于等于0的 而与其之后的元素的差是小于等于零的 而加上负数的绝对值相当于是减这个负数 （3）所以差的绝对值的和相当于可以写成 ans[i] = nums[i] * i - sum(i) - （nums[i] * n - sum(n) - nums[i] * i + sum(i)）

ans[i] = 2 * (nums[i] * i - sum(i)) + sum(n) - nums[i] * n 通过这个公式动态规划逐步求和 很容易就能得到结果

1686. 石子游戏 VI. Medium

贪心算法 解题思路 : Alice和Bob取石头的策略都是 尽可能取得自己这面分值最大的石头 而且这个石头在对方那面也尽可能大 如果这个石头在自己这面分值最大 但是在对方那面不是分值最大的 那么需要进行比较 如何比较呢？比较：每次选择的时候做比较 自己选择对方那面最大的会损失多少 以及 选择自己这面最大的对方会多得多少 两者之间做差值

其实也就是「每次都选择A[i] + B[i] 最大的那个元素」

理解：就比如Alice这面最大的是元素1 Bob那面最大的元素是2 现在轮到Alice做选择 她就要比较 A1 - A2 与 B2 - B1之间的大小 选择差值大的那个「差值越大自身获利越多」 也就是：如果（A1 - A2）- （B2 - B1）> 0 =》 （A1 + B1）- （A2 + B2）> 0 =》 选择元素1 =》 所以总结而言就是「每次都选择A[i] + B[i] 最大的那个元素」，最终比较两人手里的石子总价值

视频换背景相关论文总结

发表于 2020-12-02 更新于 2020-12-08 分类于 AI ， CV

半监督视频目标分割

半监督视频目标分割 ： 在线分割 & 离线分割

在线分割的特点 ： 速度慢 但是精度高 相当于是先训练一个通用的分割器 之后根据有标注的第一帧训练出一个专门用于这个物体的分割器

就是「先通过通用的分割器进行一次预测 之后 通过有标注的帧进行分割器的二次训练 得到最终的分割器结果」

离线分割的特点 ： 速度快 但是精度比较低

离线分割的几种方式：（1）将有标注的第一帧放进网络 相当于是将其作为一个模版进行传播 后续的分割都通过第一帧进行操作 「局限性 ： 很显然的，在整个视频中目标的形态会发生变化（就比如同一个人正对镜头和背对镜头） 所以只根据第一帧去做推断是很不合理的」
（2）根据当前处理帧的前一帧的mask预测这一帧的mask 「局限性 ： 很依赖于上一帧的效果 就比如说如果一个目标 它在某一帧被遮挡住了一部分 导致它在这一帧中没有被分割出来 那么在之后的帧中 即使它又出现了 还是不会被分割出来 由于完全依赖于前一帧的结果所以相当于这个目标失踪了」
（3）混合方法 ： 典型论文 「STMVOS Space-Time Memory 时空记忆模块 注重的是精度」 「FEEELVOS 注重的是速度 但是如果想要达到实时性的效果仍需要进行调试 且性能不如STM」

考虑到实时性 选择使用离线方法进行视频目标分割 考虑到实时性 选择使用混合方法的离线视频目标分割 下面介绍几篇典型的论文

STM （Space-Time Memory Module） 时空记忆模块

NetWork Architecture :

Methods :

（1）对于处理帧之前的帧使用编码器进行编码 生成两个特征图（分别是 「key 帧语义信息特征图」 & 「value 帧内容信息特征图」）
（2）对于当前正在处理的帧使用另一个编码器进行编码 也生成上面的两种特征图
（3）将这两部分特征图进行交互操作 送到注意力模块

Limitations :

（1）每操作一个帧 就要存储两个新的特征图 所以训练到最后 内存占用越来越多 运行速度会越来越慢
（2）操作到最后 就只能减少所存储的之前帧特征图的数量 这也就与最开始设计这个模块的初衷背道而驰了

FEEELVOS (不需要使用第一帧进行微调 且学习的过程是端到端进行的 global matching & local matching)

Methods :

（1）提出特征 : 通过DeepLabV3+(移除输出层)上再加一个embedding层得到的 => 「embedding层的向量是用来描述像素点所属类别的」 => 「两个像素对应的向量之间的距离很接近就说明两个像素点属于同一类 ； 反之说明两个像素点所属类别不同」
（2）提出两种“distacne map” => 「Global distance map」 & 「Local distance map」=> 两种map是根据（p 和 q 向量计算求得的）
两者的q都是当前处理帧的embedding向量 ， 前者的p是第一帧的embedding向量 ， 而后者的p是前一帧的embedding向量
（3）Dynamic Segmentation Head 动态分割头网络 => 四个输入部分 : 两个distance map，前一帧的预测结果，当前帧的deeplabv3+的输出特征图（不含embedding layer）
整个头网络结构由四个深度分离卷积组成，产生一个一维的feature map用来预测类别，注意这个map仅仅为一个目标预测类别logits，每个实例都会得到一个一维的feature map，将它们stack一起，使用softmax，用一个交叉熵和label训练。但即便每个实例都需要经过一次分割网络，但真正花时间的地方在于从当前帧中提取特征(Backbone Features)，作者认为这一部分是共享的，只需要进行一次就行，而显然事实也确实是如此。

1671. 得到山形数组的最少删除次数. Hard

思路：枚举每一个位置作为山形最高点，就需要计算该位置的左右两边最少需要删除多少个点。

以计算左边为例，要计算位置i左边最少删除个数delleft[i]，就要从i-1的位置向左遍历（指针记为j）直到头，每当遇到nums[j] < nums[i]的位置j，就用以下两个情况的最小值更新delleft[i]的答案
delleft[i]，表示保留原始情况，不更新
delleft[j] + i - j - 1，表示利用 delleft[j]的结果再加上夹在j和i之间所有的点都删除的总数
类似地处理每个元素右边最少删除个数delright,
之后同时遍历delleft和delright，返回最小delleft[i] + delright[i]
